# 供应商开发 · 方法论参照（思考层 + 操作层）

> **这份文件是什么**：一套成熟供应链组织在做「供应商资质确认 → 能力验证 → 建联/签约」时的**通用可迁移方法论**，抽象到「思考模型 + 具体操作」两层，不绑定任何具体公司或专案，也不含任何组织的专属产物（内部制度编号、真实供应商名称/报价、内部案例原文、内部链接一律不收录）。
>
> **怎么用**：当成一面独立的镜子——先理解「一个纪律严格的采购组织是怎么想、怎么做的」，再自己回头审视你现有做法的差异。**这是私人学习/对照用的参照，不是对外交付物的一部分。**
>
> **一句元判断**：这套方法的价值八成不在独门秘诀，而在「把几条朴素原则执行得极其严格」。下面每条操作，本质都是某个心智模型的落地。

---

## 总览：供应商全生命周期

```
推荐/寻源 → 准入(资质) → 能力验证(打样/试单) → 首单签约 → 绩效管理 → 分级 → 动态维护/退出
   L1          L2              L3                 L4          L5        L6         L7
```

每个箭头都是一道 **gate**（有明确通过标准，不达标不进下一段）。核心心态：**你不是在「买一个东西」，你是在「长期经营一个能力组合」。**

---

## 一、思考层：四组心智模型

### A. 把供应商当「要长期经营的资产组合」，不是一次采购
- **生命周期视角**：从推荐到退出每段都有动作，连「怎么优雅淘汰」都预先设计。*防*：一次性交易思维 → 无人维护、无退出机制。
- **资质会衰减**：准入设有效期、定期重评、绩效持续打分；状态是活变量。*防*：一张证书用三年。
- **不平均用力（Kraljic）**：按「重要性 × 替换难度」分层，战略级深绑给量、商品级当备胎比价。*防*：管理精力平均撒，关键的没保护、普通的过度投入。

### B. 筛选：标准先于候选，证伪先于评分
- **基线先写死**：寻源之前先定「这个品类最低必须满足什么」。*防*：被漂亮官网/话术带走，让候选人定义你的标准。
- **先找否决点**：红旗信号优先于资质本身，证伪比评分便宜。*防*：认真评估一个本该一眼刷掉的对象。
- **选对采购机制**：规格化→比价、有技术差异→招标、委外开发→谈判；「用哪种方式」本身是个决策。*防*：所有事情都用同一种采购方式。

### C. 验证：分开问「能不能」和「会不会」，买的是证据和流程不是货
- **能力 ≠ 可靠**：打样测「物理能不能做」，试单测「会不会准时、主动汇报、改稿响应」。*防*：样品惊艳、量产拉胯。
- **买证据不买承诺**：付费样、同一文件横向对比、埋雷测前处理；早期花钱是买「去风险的信息」。*防*：用免费样/一口价自我欺骗。
- **客观性靠设计**：≥2 人验厂、书面报告、签样留样当物证。*防*：单点主观、单点被公关。
- **最危险是「样品→大货」那一跳**：最强控制点压在这个交接口。*防*：大货偷偷跟打样不一样。

### D. 关系：供应商是「curated 的网络」，不是「投出去的冷开发」
- **靠长期名单 + 行业转介 + 结构化招投标**，不靠冷开发模板；寻源是策展/网络问题，不是 marketing 问题。*防*：撒网群发、质量不可控。
- **关键环节保冗余**：核心供应对象留 2–3 家备选、先小单快反再翻单。*防*：押注单一供应商，一断货就休克。

## 二、操作层：每个模型怎么落地

### L2 · 准入（资质确认）的具体做法

**(1) 品类准入基线表**——寻源前先写死，一张表，字段大致是：

| 维度 | 具体内容示例 |
|---|---|
| 强制资质 | 该品类/目标市场的法定认证（缺任一项原则上不引入，硬引入需上级加签） |
| 生产能力下限 | 核心设备类型、产能、最小起订量能否覆盖你的需求档 |
| 商务/法务下限 | 法定主体真实、经营范围匹配、无重大涉诉、财务可持续 |

**(2) 红旗一票否决清单**（先证伪，每条都配「用什么去查」）：

| 红旗 | 核查手段 |
|---|---|
| 查不到法定主体 | 营业执照 + 国家企业信用信息公示系统 / 企查查 |
| 域名年龄对不上宣称年限 | RDAP / WHOIS 查注册时间 |
| 设备宣称自相矛盾（首页说有 X 机、设备清单里没有） | 索要设备序列号/铭牌照片、在产订单视频 |
| 报价低于物理成本价 | 用材料+工艺成本反推地板价，低穿者存疑 |
| 无对目标市场的真实供货记录 | ImportYeti / Panjiva 提单反查 |
| 认证只有图标、无编号/范围 | 拿编号去发证机构官方库验证（**查无 ≠ 证伪，但举证责任在供应商**）|

> **例 · 三个红旗长什么样**
> - *设备自相矛盾*：官网首页大字「HP Indigo 数码印刷」，但 About 页设备清单只列了 UV 轮转机、没有任何 Indigo 型号 → 让对方发 Indigo 机身序列号照片；答不出＝宣称不实，降级。
> - *年限对不上*：供应商自称「18 年出口经验」，RDAP 查官网域名注册于 6 年前 → 与宣称矛盾 → 索要更早的营业执照成立日期或历史提单佐证。
> - *报价穿地板*：3"×3" 哑膜 vinyl 的材料＋覆膜＋模切物理地板价（示意）约 $0.04–0.05/张（500 档），某家报 $0.008/张 → 要么是「起批噱头价」要么偷工，追问「这是哪个数量档、含不含模费的真实到手价」。

**(3) 准入评分卡**——不是过/不过二元，是分数决定动作：
- 维度：资质合规、生产/技术能力、质量体系、商务成本、交付能力、服务响应。
- 逻辑：高分直接发盘 / 中段补尽调 / 低分弃（例：≥75 发盘、55–74 补尽调、<55 弃——阈值按品类校准）。

**(4) 准入的时效**：有效期（如 1 年）；连续 N 个月（如 36）无采购活动自动失效；战略级供应商年度重审。

### L3 · 能力验证（打样 + 验厂 + 试单）的具体做法

这是整套里最需要「工程化」的一段。分三条并行。

**(a) 打样协议**
- **付费样优先**：免费样往往寄库存成品，测不出按你文件的定制能力；样费通常抵货款。
- **同一文件横向可比**：同一个设计文件发 2–3 家，才能横向对比。
- **埋雷测前处理能力**：给一个**故意埋了 3 个问题的测试文件**，看对方能否逐项指出。印刷类可埋的雷：
  - 出血不足（bleed < 3mm）
  - 刀线/切割路径未闭合（open cutline path）
  - 色彩模式错误（发了 RGB 而非 CMYK / 专色未转换）
  - 缺刀线图层或刀线与图案不对齐
  - 图片解析度过低（<150–300 dpi）

> **例 · 埋雷文件与「及格的回覆」**
> 你发一个测试稿，故意留三个雷：① 出血只留 1mm（标准 3mm）② 刀线是开放路径、没闭合 ③ 主图是 RGB、150 dpi。
> - **及格**（前处理能力真实）：「稿件出血不足，已帮您补到 3mm；刀线未闭合已连接成完整路径；主图为 RGB 且解析度偏低，建议转 CMYK 并提到 300 dpi，否则成品会偏色发虚。」——三个雷全指出。
> - **存疑**：闷头回一个价、只字不提文件问题 → prepress 能力打问号，量产大概率出色差/切歪。

- **样品实测清单**（印刷/贴纸类）：色差（对色卡/Pantone）、刀线精度（≤0.5mm）、胶性/背胶残留、防水/耐刮、材质手感与宣称是否一致。

**(b) 打样阶段序列**（有模具/开版的品类，越往后越不可逆）
| 阶段 | 可改什么 | 典型周期 |
|---|---|---|
| 手板/3D 打印样 / 数码打样 | 外观、配色可反复改 | 3–5 工作日 |
| 开模/开版 | 一旦开始，外观不可改 | 数码贴纸无此步；柔印/凸印开版 3–5 周 |
| 产前样/大货样 | 与大货完全一致，仅微调颜色 | 与大货同工艺 |

**关键动作——封样/签样**：产前样由**双方在样品上签字留底**，作为大货验收的物证基准；大额订单三方（需求方/采购/供应商）都留样。**这是防「大货≠打样」最经典翻车的唯一物证。**

> **例 · 签样为什么能救你**
> 场景：5,000 张哑膜白 vinyl，打样批准的是哑面、纯白。大货到手发现表面偏亮、白色偏灰——工厂为压成本换了背膜和白墨。
> - **有签样**：双方签过字、你手上留着封样 → 直接拿签样对比，物证在手，拒收整批 + 要求免费重做。
> - **没签样**：变成「你说偏灰、他说正常」的扯皮，最后多半只能自己吞。

**(c) 验厂**
- **客观性设计**：≥2 人同行、结束出书面报告存档。
- **现场核查动作**：走一遍产线、拍设备铭牌、看在产订单是否与宣称产能一致、查证照有效期（用具体规则如「到期日 > 30 天」）、查对上游采购合同、追溯进料批次。
- **验厂评分卡**（商家/供应商准入常见 6 模块，各按权重打分）：① 证照资质合规 ② 上游供应商管控 ③ 入库/进货品质管控 ④ 场所/仓库安全 ⑤ 发货及售后管控 ⑥ 运营能力。
- **轻量层**：年采购额未到阈值前，可用**视频验厂**（走车间 + 拍铭牌 + 看在产）替代付费实地验厂。

> **例 · 视频验厂当场露馅**
> 让对方举手机走到数码印刷机前、拍一下机身铭牌和正在跑的订单。结果对方镜头一直躲、只拍成品堆，说「机器在隔壁不方便拍」 → 大概率没有自称的设备（可能是找同行代工）→ 要求补拍指定特写，给不出就降级或剔除。

**(d) 首单/试单——测的是「人」不是「货」**
- 小单（如 100 张）真正在测：改稿响应速度、状态是否主动汇报、交期兑现、与封样一致性。
- 心态：**买流程数据，不是买货。**

**(e) 到货检验**
- 抽检标准：按 AQL 抽样；小批量（如 ≤1000）可全检外观 + 抽检功能；**拆箱录像**留证。
- 合格率闸门（如 ≥97%）不达标触发整改/退货。

### L4 & L5–L7 · 签约、绩效、分级、退出

**(1) 询价/建联机制**
- **长名单 → 短名单**：长名单来源＝库内长期合作 + 行业优质供应商 + 转介；一次发 5–8 家（预期回复 50–70%，最终拿 3–5 份可比报价）。
- **RFQ 必须逐行（line-item）**，逼出成本结构。完整字段：分档数量的单价、MOQ、模费/开版费、样费 + 交期、生产交期、文件规格要求、关键工艺能力、认证（编号 + 范围：成品 vs 原料）、付款条款、贸易条款（DDP 等）、报价有效期、**保密承诺（客户稿件不得留底/复用/转售）**。

> **例 · 「及格的逐行报价」vs「偷懒的回覆」**
> 你问 3"×3" 哑膜 die-cut vinyl 的分档价，**及格**的回覆是一张表（数字为示意）：
>
> | 项目 | 100 张 | 500 张 | 1,000 张 | 5,000 张 |
> |---|---|---|---|---|
> | 单价 (USD) | 0.42 | 0.11 | 0.07 | 0.045 |
> | 模费/开版 | $25（一次性，翻单免） | | | |
> | 样费 / 交期 | $30 / 3 天 | | | |
> | 生产交期 | 5–7 天 | | 7–9 天 | 10–12 天 |
> | 认证 | FSC-C 面材编号可提供、食品接触 FDA 迁移报告可提供 | | | |
> | 付款 | 30% 定金 / 70% 发货前，走 Trade Assurance | | | |
>
> **偷懒的回覆**：「3 寸 $0.1 起，可以做，加微信聊」——没有分档、没有模费、没有认证编号、没有交期 → 无法比价，也测不出专业度，直接降优先级。

- **通道纪律**：正式 RFQ 走邮件留痕；日常跟进走 IM（微信/WhatsApp，响应快一个量级）；**关键报价条款务必留一份在有留痕的通道里**。

**(2) 采购机制三选一**
| 机制 | 适用 | 怎么跑 |
|---|---|---|
| 招标 | 技术/方案有差异 | brief + 技术评分表 + 述标 + 评委打分 |
| 比价 | 规格化、无技术差异 | 短名单须先过能力 gate，再低价者胜 |
| 谈判 | 委外开发/定向 | sourcing 后定向谈 |

**(3) PI（形式发票）9 要素**：买卖双方主体、产品规格、数量/单价、**质量条款（挂钩签样）**、包装、交期、付款条款、贸易/物流条款、售后/返工条款。
**付款纪律**：永不 100% 前 TT；定金/尾款结构；有平台则用 Trade Assurance 类托管。

**(4) 绩效管理（上线后持续）**
- 维度：质量（不良率/PPM）、交付（准时率 OTD%）、服务/响应、商务/成本、合规。
- 节奏：季度/年度业务回顾（QBR/YBR）反馈。
- 淘汰：绩效 < 阈值（如 60 分）且整改未达标即触发；欺诈/失信/泄密进黑名单。

**(5) 分级（Kraljic 2×2）**
```
              供应风险 高
                 │
   瓶颈 Bottleneck │ 战略 Strategic
   (保供、找替代)  │ (深绑、联合改进、长约)
 ─────────────────┼───────────────── 重要性/金额 高
   非关键 Routine  │ 杠杆 Leverage
   (自动化、省心)  │ (充分竞价、压价)
                 │
              供应风险 低
```
- ABC 人群形态（约数）：战略 A 类 ~1%、优秀 B 类 ~10%、合格 C 类 ~50–60%。
- 落地：战略级留 2–3 家备选、给量深绑；商品级充分比价、当备胎；先小单快反（首单 500–1000 测试），数据验证再翻单。

> **例 · 把贴纸采购项摆进四象限**
> - *杠杆象限*（量大、可替换厂多）：主力 die-cut 哑膜 vinyl → 充分比价、每季度重新压价。
> - *瓶颈象限*（重要但只有一两家能做）：某种特殊全息/镭射材料 → 别只靠一家，主动找第二供应商保供。
> - *战略象限*（量大又难替换）：某家同时给你最好的价 + 最稳的品质 + 能承接设计协助 → 长约深绑、联合改进。
> - *非关键*：一次性的模切刀模 → 省心就好，别花管理精力。

## 三、情景举例（互联网公司定制实体商品 · 均为虚构化名示意）

> 以下三个场景是**虚构组合示意**：公司、产品、数字都是化名/示意，但采购品类、供应商行业、流程节点尽量贴近真实行业实践。用来看「同一套方法在不同品类里怎么跑」。

### 场景 A：短视频平台给虚拟形象做亚克力周边（潮玩/亚克力品类）

**需求**：某短视频平台（下称 A 厂）给自家虚拟形象做 3,000 套亚克力立牌 + 金属徽章，当创作者大会伴手礼，大会日期是硬 deadline。

- **准入基线**：亚克力切割 / UV 打印或移印色准、金属徽章咬花电镀、环保（皮肤/儿童接触则更严）、能否在 deadline 前交。
- **长名单**：从潮玩/亚克力产业带（如广东东莞、汕头澄海一带）+ 同行转介凑 6–8 家。
- **红旗筛**：某厂官网写「自有 3D 打印手板 + 移印线」，但车间视频只有切割机、手板说「合作方做」→ 手板外包＝改稿多绕一轮，deadline 项目降级。
- **打样**：亚克力先出激光切割/3D 手板（3–5 天）看尺寸与边缘，移印/UV 打样重点看**色准能不能对上虚拟形象的品牌色**（二次元周边最容易翻的点）。
- **封样（deadline 项目的命门）**：产前样双方签字，大会前每 2 天催一次进度。
- **Kraljic**：普通亚克力立牌＝杠杆象限（多家能做、充分比价）；某款镭射/渐变特效材料只有一两家稳定做＝瓶颈象限（别单点、找第二家保供）。
- **典型翻车**：大货移印比签样偏色半个色阶、大会在即。有签样 → 拿去压工厂免费重印赶工；没签样 → 只能含泪发。

### 场景 B：跨境电商团队白牌选品（3C 配件品类）

**需求**：某跨境电商团队（下称 B 团队）在美国站上一款白牌磁吸手机支架，从深圳/东莞 3C 产业带找厂。

- **红旗筛**：某厂自称「常年供货美国大牌」，但 ImportYeti/Panjiva 查不到它对美真实提单 → 让它拿客户/提单佐证，拿不出就存疑。
- **认证**：磁吸/带电子件的品类在美国可能涉及 FCC/UL/CPC，拿证书编号去官方库验真，别信官网图标。
- **小单快反**：首单只下 500–1,000 个测链路（不一上来压 1 万），用退货率/差评验证再翻单。
- **备援**：核心 SKU 留 2–3 家备选，旺季防单点断货。

### 场景 C：品牌市场团队做包装标签（数码印刷/不干胶品类）

**需求**：某消费品牌市场团队（下称 C 团队）给新品做不干胶标签 + 包装贴纸，要上美国货架。

- **RFQ 逐行**：分档单价 + 模费 + 样费/交期 + 认证编号，一次发 5–8 家收 3–5 份可比报价。
- **埋雷测前处理**：发一个出血不足 + 刀线未闭合 + RGB 低清的测试稿，看哪家主动指出（详见 L3 埋雷例）。
- **认证**：食品接触类要 FDA 迁移测试报告、环保诉求要 FSC/可堆肥证书编号 + 覆盖的面材范围。
- **付款纪律**：30/70 定金尾款、走 Trade Assurance，永不 100% 前 TT。

## 四、常见翻车模式 → 对应控制（快速对照表）

| 翻车模式 | 根因 | 控制动作 |
|---|---|---|
| 大货与打样不一致，验收才发现 | 无物证基准 | 产前样双方签样 + 三方留样；硬 deadline 每 1–2 天催、上机前最后确认 |
| 免费样过了、量产拉胯 | 免费样是库存成品 | 付费样 + 同一文件横向对比 |
| 认证宣称查无实据 | 只看图标不看编号 | 编号去发证机构官方库验；举证责任在供应商 |
| 好看的官网 = 好供应商？ | 标准被候选定义 | 品类准入基线先于寻源；红旗一票否决先于评分 |
| 单一供应商断货休克 | 押注单点 | 核心 SKU 2–3 家备选、主备分单 |
| 样品好但沟通/交期灾难 | 只验了货没验人 | 首单试单测软指标（响应/汇报/交期）|
| 验厂被一个人「公关」掉 | 单点主观 | ≥2 人验厂 + 书面报告存档 |
| 证书早过期还在用 | 准入无时效 | 准入设有效期 + 到期重审 + 无单自动失效 |
