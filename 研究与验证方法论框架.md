# 研究与验证方法论框架

> 从 Custom Sticker 项目全程实操中拆解（2026-06-26 ~ 07-03），每条方法都附本项目的真实用例。适用对象：任何「新品类 × 新市场 × 供应链」型研究命题。

## 一页骨架

```
┌─ 主循环（每轮研究都走一遍）─────────────────────────────┐
│                                                        │
│  ① 拆解命题 ──→ ② 并行取证 ──→ ③ 证据分级 ──→ ④ 对抗验证 │
│       ↑                                        │       │
│       └──── ⑥ 批评与缺口 ←──── ⑤ 综合成方案 ←───┘       │
│                                                        │
└─ 贯穿纪律：诚实边界 · 可复现 · 时间锚定 · 面向行动 ──────┘
```

一句话版本：**把大问题拆成可证伪的承重论断，多路并行取证并分级标注，用「默认证伪」的对抗机制杀掉似是而非的结论，再把幸存结论串成每个动作都可回溯的方案；每轮以「下一步动作 + gate」收尾，循环由完整性批评驱动。**

---

## ① 拆解命题：先决定「什么值得验证」

| 方法 | 要点 | 本项目用例 |
|---|---|---|
| 命题 → 子问题矩阵 | 按题目要求逐条铺开（市场/竞品/规格/供应链/成本/渠道），做覆盖度对照表，防漏答 | 作业需求逐条核对，曾查出「不同数量对应价格」「小批量 vs 规模化成本结构」两处薄弱 |
| **承重论断识别** | 区分「结论压在上面的数字」和「装饰性事实」——验证预算只花给前者 | 白空间论证轮只挑了 16 条承重论断做对抗核查，而不是核查所有数字 |
| 预写证伪条件 | 每个论断先写清：什么证据能证实、什么能证伪（不可证伪的表述要改写） | 「环保缺口是白空间」被改写为可查证的五条件交集（认证可堆肥×FDA×定制×<100张×自助），然后逐家核 |
| 假设排序 | 按「致死性 × 验证成本 × 周期」排验证顺序，零成本先验 | H1–H4 判死线表：4over 实价（$0/1周）排在投流测试（$1500/60天）之前 |

## ② 并行取证：多路盲扫 + 证据优先级

| 方法 | 要点 | 本项目用例 |
|---|---|---|
| 多模态并行 | 多路同时扫，**每路盲于其他路**（按平台/按内容/按主体/按时间各一路），单一角度必有盲区 | 供应商扩池 6 路：Alibaba / MIC·GS / 独立官网 / 食品级定向 / 环保定向 / BRCGS 目录直查 |
| 证据优先级金字塔 | 权威库直查 > 一手页面实抓（带日期/截图）> 行为证据 > 二手行研 > 媒体转述 > 对象自述 | 关税走 USITC 税则库而非新闻；竞品价格用配置页实拍截图（标注 2026-07-02） |
| 行为证据 > 意愿证据 | 调研问卷说「愿意付费」不算数，看真实交易/真实行为 | Trivium「82% 愿为可持续付费」被 Etsy 类目 GMV 仅 $1–3M、Reddit 抽样 6:1 自印所推翻 |
| **活体检验** | 断言某模式已死之前，先看现存活体怎么活着 | 「中国直邮已死」经 Vograce/GS-JJ 拆解修正为「免税价差已死」——在位者把关税打包进 DDP 运费继续活 |
| 负面证据也是证据 | 搜索量小、目录查无、请求票数低，本身就是天花板/缺口的量化信号（但要声明口径限制） | 细分头部词月搜仅 30–420 次；Sticker Mule 反馈板对口请求仅 137 票/7 年 |

## ③ 证据分级：标注纪律

| 规则 | 要点 | 本项目用例 |
|---|---|---|
| 三级标签 | 每个承重数字标 **确证 / 估算 / 待验证**，全文无裸奔数字 | Grand View $16 亿标「确证」；Sticker Mule 营收 $1–5 亿标「估算·口径分歧大仅作量级参考」 |
| 区间不假精确 | 不确定的给区间 + 中枢，禁止编出小数点 | TAM $90万–1,880万（中枢 ~$450万） |
| 外推必须晒假设 | 乘数、口径、放大逻辑写在数字旁边，让读者能自行折价 | 「TX 1.2 万家外推：3–5 倍乘数 = 人口等比放大 + 两项立法假设」 |
| **分层不可混池** | 不同验证深度的对象分层管理、显式声明不可直接对比 | 实核 12 家 vs 线上初筛 24 家分段展示，扩充池分数注明「不可与实核直接对比」 |

## ④ 对抗验证：核心引擎（七件武器）

| 武器 | 机制 | 本项目用例 |
|---|---|---|
| **双视角核查** | 视角 A 追原始来源核口径与权威性（警惕付费通稿）；视角 B **不看原来源**独立三角验证；**拿不准默认证伪** | 16 条承重论断逐条双视角；「$4.84B 市场规模」被证伪剔除 |
| **权威库直查 + 阳性对照** | 关键事实绕过搜索引擎转述直查数据库；查「没有」之前先证明「查得到已知存在的」（排除检索盲区） | BPI 库查可堆肥认证时先用在证的 Sinclair PLU 贴做阳性对照，才敢下「全美在证为零」 |
| 红旗体系（否决/扣分两级） | 一票否决：无法定公司名、设备宣称自相矛盾、报价低于物理成本；扣分：域名年龄 vs 自称历史、认证无编号、主业错配 | LabelPrintingChina 三重红旗出局；华鑫美「首页称 Indigo 设备表无」降为有条件 |
| 内部一致性审计 | 把正文、模型、附表的同一数字跨处对账——自相矛盾比外部错误更致命 | 揪出 CAC 双账本（模型 $8 vs GTM 章 $80–150）、同一笔百张订单三个售价 |
| **红队评审 + 默认驳回复核** | 多 persona（研究/产品/GTM/供应链/审计/初筛/负责人）独立批判 → 合并去重 → 每条过「默认驳回」的复核（批评所指是否真实存在/是否范围内/修复收益是否配得上成本）——防批评通胀 | 面试官评审团：38 条原始发现 → 14 条合并 → 复核后 24 处修订落地 |
| 完整性批评 | 专门问「什么没做」：没扫的模态、没验的声称、没读的来源——产出下一轮工作清单 | 「已知局限」成为报告一等公民：软件层未扫（后已补）、搜索量口径薄弱、双侧报价未闭环 |
| 「查无 ≠ 证伪」 | 数据库查不到 ≠ 对方没有（持证主体名不同/库不全），给出复议路径而不是下死刑 | 中国厂常以中文法定名持证——复议以对方提交证书编号、发证机构官网可查为准 |

## ⑤ 综合成方案：可追溯 + 面向行动

| 方法 | 要点 | 本项目用例 |
|---|---|---|
| **推导主线表** | 「研究发现（出处）→ 判断 → 方案动作（去处）」三列表；任何方案动作找不到依据行 = 拍脑袋，应质疑 | 报告 0.5 节 10 行主线；B/C 每个模块标「← 研究依据」 |
| 判定要正面 | 给 build/kill 明判 + 前提 gate + 12 个月量化目标，不把一堆分析留给读者自己拼结论 | 0.9 六十秒判定：做，前提三 gate 60 天内全过，任一失败即退出或转向 |
| 不确定性流程化 | 关键未知数不是脚注，而是转成带判死阈值和失败分支的验证动作 | 4over 实价 → H1（>$47 则砍档）；复购率 → H4（<15% 则 no-go） |
| 结论可复现 | 模型数字必须能从表列假设逐行算出；算不出来的旧结论作废重算 | 「100 张净利 0–8%」无法复现→作废，按 BASE 曲线重算为 ~58%（经营口径）并读回公式验证 |
| 样本不足时换 proxy 判据 | 预算买不到统计显著时，明说测不出什么、改测漏斗前级指标 | 首月 $1,500 只能买 10–19 客——明写「首月不以 ROAS 为判据」，改测 CPC→CTR→配置器启动率阶梯 |
| 操作性交付（playbook ≠ 清单） | 给零基础可照做的 stage-gate：每阶段有目标/动作/通过标准/踩坑；「当前位置」标注在图上 | E 节供应链 0→8 阶段 + E0.5 POD SOP；每条红旗都来自本项目实核踩坑 |

## ⑥ 贯穿纪律（元规则，违反任何一条都会污染全链）

1. **诚实边界**：没做过的验证不声称做过。用例：评审团草稿写「已用手机 4G 验证链接」，因未实际执行被改成「提交前必做清单」；原型仅实现步骤①，在报告里如实声明而非宣称「已实现」。
2. **时间锚定**：每个数据标采集日期；政策类事实记生效日与到期日；用 Wayback 做前后对比切点。用例：关税 07-24 到期重算窗口；de minimis 两个 EO 生效日作为在位者条款对比切点。
3. **面向行动收尾**：每轮研究的产出不是「结论」而是「下一步动作 + gate + 截止时间」。用例：E2 五条下一步逐条挂 playbook 阶段号。
4. **同步一致性**：正文、数据源 CSV、线上表格、原型文案四处联动改，改完做读回校验（公式重算值 vs 正文数字、截图锚点位置）。用例：hero 文案矛盾同轮修掉并 curl 验证线上生效。

## ⑦ AI 编排模式（怎么用多 agent 实现上述框架）

| 模式 | 结构 | 关键设计 |
|---|---|---|
| 研究轮 | 并行扫描（各路互盲）→ 承重论断分诊 → 双视角对抗验证 → 综合 + 完整性批评 → 缺口补数 | 验证者与发现者**必须分离**（不核自己找的东西）；验证默认证伪 |
| 评审轮 | 多 persona 批判 → 合并去重定级 → 默认驳回复核 → 需要新数据的才补研究 → 带锚点的成品级修订 | persona **多样性 > 数量**；复核三问（真实存在？范围内？收益配成本？） |
| 扩池轮 | 按能力缺口定向猎源（各路互盲）→ 代码级去重 → 逐家存在性/红旗核验 → 评分 → 分层入池 | 硬性反编造规则：每家必须来自本轮实际看到的页面；核验深度如实分层 |
| 人机分工 | AI：扫描、交叉验证、建模、生成、同步；人：打样实测、真实谈判、需实体资质的注册、扫码授权 | 线下闭环点**显式列出**（¥200 打样、4over 转售证书、权限设置），不假装 AI 能替 |

## ⑧ 复用清单：换一个新品类（如定制纸杯/环保餐盒），第一天做什么

1. 用「命题 → 子问题矩阵」铺开题目，标出承重论断候选（约 10–20 条）。
2. 起并行扫描：竞品量价（带截图）、供应链平台粗名单、关税税则（权威库直查 HTS）、需求侧行为证据（Etsy/Reddit/搜索量）。
3. 所有数字进「确证/估算/待验证」三级标注，建可调成本模型（假设行与结果行分离，公式可读回验证）。
4. 对承重论断跑双视角对抗核查（默认证伪）+ 权威库直查（带阳性对照）。
5. 供应链走 stage-gate playbook（本项目 E 节品类无关，可直接套用：寻源渠道、红旗清单、评分卡、RFQ 模板、打样 checklist、封样条款、物流决策规则全部可复用，只换品类参数）。
6. 综合时先写推导主线表和 0.9 判定框，再填正文——**先有骨架再有肉，防止「各环节独立不串联」**。
7. 上红队评审轮（含数字审计 persona 专查跨章节对账），修订后声明已知局限。
8. 每轮收尾输出：下一步动作 + gate + 判死线。
